## Findings

- **The discovery chain works under a weak model.** The `--log-ids` flag on `coach
  athlete-workouts` surfaces `savedWorkoutSetExerciseId` cleanly, `exercise resolve` gives the
  replacement id, and the `swap-exercise` help text names exactly where each id comes from. No
  per-surface help fix needed.
- **Field name `prescribedExerciseId` was ambiguous** (write run). After a swap the athlete *is*
  prescribed the new exercise, so "prescribed" read as the current state rather than the untouched
  team original. **Fixed**: renamed to `originalTeamExerciseId` in `SwapExerciseResult`
  (`packages/js/src/athlete.ts`) and its test. Shared field, so both CLI and MCP benefit.
- **cr3 took 4 commands** (vs the 2–3 ideal) — mild exploration of the help text before acting,
  but no reported confusion and the right answer. Not worth a change.
