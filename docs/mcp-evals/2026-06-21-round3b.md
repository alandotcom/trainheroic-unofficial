# MCP eval — 2026-06-21 (round 3b): coach athlete_training

Follow-up to round 3. Round 3 concluded the coach's cross-exercise discovery gap had no live
endpoint, based on `curl` probing alone. That was wrong. Logging into the coach web app
(`coachapp.trainheroic.com`) with the agent-browser skill and watching its network calls surfaced
the endpoint the UI uses, and it is a clean live-API call that both the local and hosted coach
surfaces can share.

## What the browser found

Opening a roster athlete's calendar (`coachapp.trainheroic.com/athlete/{id}/{year}/{month}`) fires
`GET /2.0/coach/athlete/calendar/summary/{athleteId}/{year}/{month}/{n}`. It returns that athlete's
month of sessions: per session a `workout_title`, the `logged`/`completed` flags, `rpe`,
`session_duration`, `notes`, and `sets[].exercises[]` each with `exercise_id`, `title`, and an
`abr` set-summary string ("5 x 2 @ 205 lb"). The trailing path segment is required but ignored
(1, 7, and 31 all return the same month). There is no per-session date in the payload; sessions are
in calendar order within the month.

The coach `/3.0/coach/athlete/numbers/{org}/{athlete}` "stats" call was a red herring: it returns
the whole 2,000+ exercise catalog with `pr_value`/`hasHistory` null for this athlete (it holds
coach-set goals and working maxes, not logged PRs).

## Shipped

- **`athlete_training(athleteId, year, month)`** (coach core tool) over the calendar-summary
  endpoint, with a `presentCoachAthleteTraining` presenter in `js` (one row per session: title,
  `logged`, rpe, duration, notes, and the exercises performed with their `abr` summary). Defined in
  `core`, so the local stdio coach server and the hosted worker both get it. Fixture-backed test.
- **`athlete_lift_history` description** now points at `athlete_training` as the discovery step:
  read the month to see exactly which exercises the athlete performed (the real one the program
  used, e.g. "Clean & Jerk" not "Clean"), then pull those lifts' PRs — instead of guessing names.
- `scripts/mcp-eval.sh` coach read-whitelist includes `athlete_training`.
