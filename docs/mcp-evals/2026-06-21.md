The coach surface is navigable (all reached yes) but has two confusion-3 traps and a cluster of
analytics-key issues:

- **`list_programs` dead end** ‚Äî it returns only standalone programs ([] here); the real program
  lives as a `group_program` on the team. An agent reaching for the obvious tool concludes "no
  programs."
- **`analytics_categories` keys ‚âÝ `analytics_query` metric enum** ‚Äî two agents tried to feed the
  catalog's raw names (`training-summary-team`, `readiness-survey-athlete`) into `analytics_query`
  and had to guess the real enum values.
- **No team-wide training summary** ‚Äî team volume requires querying `training-summary-athlete`
  per athlete; the agent first had to discover athlete ids.
- **`list_teams` athleteIds only contained the coach**, not the members; `member_count` /
  `athlete_count` / `athleteIds` disagree. No per-team athlete list.
- Minor: `can_edit` (the custom-exercise signal) is undocumented; `messaging_read` `limit` has no
  documented default; readiness sub-score direction (is stress inverted?) is unexplained.

## Shipped this session

- `athlete_workouts`: added `loggedOnly` and `limit` (SDK `selectWorkouts`, core tool, CLI
  `--logged-only`/`--limit`). `loggedOnly` cuts a week from 15 workouts to the 1 that was logged.
- Description fixes: `athlete_exercises` now cross-references `athlete_personal_records`;
  `athlete_personal_records` and `athlete_exercise_stats` describe their roles and the id
  prerequisite; `analytics_query` says its enum is the catalog and notes there is no team
  training summary (query per athlete via `list_athletes` ids); `analytics_categories` warns its
  keys do not match the query enum.
- (Earlier this session) the `logged`/`performed` merge in `presentAthleteWorkout` and the
  warehouse mirror (migration 0004).
- Destructive-gate coverage: `confirm-gate.test.ts` now asserts every gated coach tool
  (`athlete_invite`, `athlete_archive`, `team_delete`, `team_code_delete`, `message_send`,
  `message_delete`, `workout_publish`, `session_remove`, `session_unpublish`) fails closed when
  elicitation is declined or unavailable, and only hits the API with `confirm:true` (which never
  elicits). Mock client/elicitation, so no live writes.
- Eval tooling: the `mcp-eval` skill now drives a custom `mcp-eval-runner` subagent and supports
  both `trainheroic-local` (athlete) and `trainheroic-local-coach` servers.
